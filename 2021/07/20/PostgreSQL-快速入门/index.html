pgsql简介

排名靠前的关系型数据库，与Oracle同根同源，所以语法相似兼容

Data Definition Language (create/drop/truncate)
Data Control Language (grant/revoke)
Data Manipulate Language (insert/update/delete)
Transaction Control Language (commit/rollback/savepoint)

  • postmaster主进程派生其它子进程
  • checkpointer子进程检查点进程，将某个时间点之前shared Buffer中的脏数据都刷新到磁盘，类比Oracle DBWR进程。
  • walwriter子进程写日志进程，类比Oracle LGWR
  • 垃圾回收进程 autovacuum
  • 统计信息收集进程stats collector
  • 逻辑复制进程logical replication laucher

认证配置

PostgreSQL以哪个用户安装数据库软件，那么内部会创建同名的用户为数据库默认管理员，这就是为什么切换到安装用户shell直接psql就可以登录数据库。

配置文件/usr/local/pgsql/13/data/pg_hba.conf
hba： host basic auth 基于主机的权限认证

  • trust 表示可信，不需要认证
  • md5 表示需要密码认证，但是保存和通信是明文容易被嗅探
  • scram-sha-256 表示需要密码认证，并且通信是加密传输，最为安全的方式

启停管理中的模式

  • smart模式，相当于oracle transction;quit after all clients have disconnected
  • fast模式，关闭数据库过程中会回滚相关未提交的事务，下次启动数据库时无需instance recovery,类似于oracle shutdown immediate模式
  • immediate模式类似于oracle中的abort, quit without complete shutdown; will lead to revocery on restart
